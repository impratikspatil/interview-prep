The List Family

ArrayList
Object[] array
+Random access & iteration, O(1) get
−Insert / delete in the middle is O(n)
·Default capacity 10, grows 1.5×

+Insert / delete at head & tail, O(1)
−Random access is O(n) traversal
·Implements List and Deque

Set · no duplicates

The Set Family

HashSet
backed by HashMap
Fastest — O(1) add, contains, remove. No order guarantee. Allows one null.
LinkedHashSet
HashMap + linked list
Same O(1) speed, plus keeps insertion order. Slight overhead for the linked list.
TreeSet
red-black tree
Sorted, O(log n). Adds first / last / floor / ceiling. No nulls.

Java Collections Framework08 / 19
Queue / Deque · ends-first access

The Queue & Deque Family

ArrayDeque
resizable circular array
O(1) at both ends, great cache locality. The docs recommend it over Stack and LinkedList. No nulls.
PriorityQueue
binary min-heap
O(log n) insert / remove, O(1) peek. poll() returns the smallest. Not thread-safe.
BlockingQueue
thread-safe interface
Blocks producers when full, consumers when empty. The backbone of concurrent pipelines.

Java Collections Framework10 / 19
Map<K,V> · key → value

The Map Family

HashMap
hash table
O(1) everything, no order. One null key, many null values. Not thread-safe.
LinkedHashMap
hash table + linked list
Keeps insertion (or access) order. Access-order mode → an LRU cache.
TreeMap
red-black tree
Sorted keys, O(log n). NavigableMap range queries. No null keys.
Hashtable
synchronized (legacy)
Locks the whole table on every call. Avoid in new code.
ConcurrentHashMap
CAS + per-bucket locks
Thread-safe, lock-free reads. The modern replacement for Hashtable.

Iteration Behaviour

Fail-Fast vs Fail-Safe

Fail-Fast
throws ConcurrentModificationException
EXAMPLES
ArrayList, HashMap, HashSet
MECHANISM
Checks a modCount field on each step.
MEMORY
Efficient — no copy.
USE FOR
Single-threaded iteration.
Fail-Safe
iterates a snapshot, no exception
EXAMPLES
ConcurrentHashMap,
CopyOnWriteArrayList
MECHANISM
Iterates over a copy / snapshot.
MEMORY
Higher — maintains a copy.
USE FOR
Concurrent iteration.
Java Collections Framework15 / 19
Ordering

Comparable vs Comparator

Comparable
Implemented by the class itself
One natural ordering
compareTo(T o)
e.g. String, Integer, Date
Comparator
External to the class
Multiple orderings possible
compare(T o1, T o2)
Can't modify class? Use this.
Comparator<Student> byName = Comparator.comparing(s -> s.name);
Comparator<Student> byGrade = Comparator.comparingInt((Student s) -> s.grade).reversed();
Comparator<Student> both = byGrade.thenComparing(s -> s.name);
TreeMap and TreeSet accept a Comparator right in their constructor.
